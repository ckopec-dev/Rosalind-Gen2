# Rosalind Problem: Find_a_Profile-most_Probable_k-mer_in_a_String

## Problem Understanding

Given a string, a k-mer length, and a profile matrix, we need to find the k-mer that has the highest probability of being generated by the profile matrix.

## Solution Approach

1. Generate all possible k-mers from the input string
2. Calculate the probability of each k-mer according to the profile matrix
3. Return the k-mer with the highest probability

## Ruby Implementation

```ruby
def profile_most_probable_kmer(dna, k, profile)
  # Generate all possible k-mers from the DNA string
  kmers = []
  (0..dna.length - k).each do |i|
    kmers << dna[i, k]
  end
  
  # Find the k-mer with maximum probability
  max_prob = -1.0
  most_probable_kmer = ""
  
  kmers.each do |kmer|
    prob = calculate_probability(kmer, profile)
    if prob > max_prob
      max_prob = prob
      most_probable_kmer = kmer
    end
  end
  
  most_probable_kmer
end

def calculate_probability(kmer, profile)
  probability = 1.0
  
  kmer.chars.each_with_index do |nucleotide, i|
    case nucleotide
    when 'A'
      probability *= profile[0][i]
    when 'C'
      probability *= profile[1][i]
    when 'G'
      probability *= profile[2][i]
    when 'T'
      probability *= profile[3][i]
    end
  end
  
  probability
end
```

## Key Points

1. **Input Format**: The profile matrix is a 4×N matrix where rows represent nucleotides A, C, G, T and columns represent positions
2. **Probability Calculation**: For each k-mer, multiply the probabilities of each nucleotide at each position
3. **Edge Cases**: Handle cases where probabilities might be zero or very small
4. **Time Complexity**: O(N×K) where N is the length of DNA string and K is the k-mer length

The solution correctly identifies the profile-most probable k-mer by calculating the probability for each possible k-mer and selecting the one with the highest probability.
